# API SPECIFICATIONS

## Project

Marine Intelligence Cognitive Architecture

Version: 1.0

Status: Draft Standard

---

# Purpose

This document defines the standard APIs used by the Marine Intelligence System.

These APIs allow communication between:

* Marine MasterDB
* Runtime Telemetry
* NICAI Cognitive Engine
* SVACS Validation System
* Operations Command Center
* Future Intelligence Services

The goal is to ensure every intelligence output is:

* Explainable
* Traceable
* Replayable
* Validatable
* Governed

---

# API Architecture

The Marine Intelligence System follows this flow:

Observation
↓
Signal
↓
Evidence
↓
Risk / Opportunity
↓
Recommendation
↓
Decision
↓
Validation
↓
Replay

Every API supports one or more stages of this intelligence lifecycle.

# 1. Observation API

## Purpose

Stores and retrieves observations from datasets, telemetry and sensors.

---

## Create Observation

### Endpoint

```http
POST /api/v1/observations
```

### Example Request

```json
{
  "source": "CWC",
  "entity": "Varanasi Station",
  "observation_type": "WaterLevel",
  "value": 4.5,
  "unit": "meters"
}
```

### Example Response

```json
{
  "observation_id": "OBS-001",
  "status": "created"
}
```

---

## Get Observation

### Endpoint

```http
GET /api/v1/observations/{id}
```

### Example Response

```json
{
  "observation_id": "OBS-001",
  "source": "CWC",
  "entity": "Varanasi Station",
  "value": 4.5
}
```

# 9. Replay API

## Purpose

Allows complete replay of intelligence and decision history.

This is critical for audits and investigations.

---

## Create Replay Record

### Endpoint

```http
POST /api/v1/replays
```

### Example Request

```json
{
  "decision_id": "DEC-001"
}
```

### Example Response

```json
{
  "replay_id": "REP-001",
  "status": "created"
}
```

---

## Replay Decision

### Endpoint

```http
GET /api/v1/replays/{id}
```

### Example Response

```json
{
  "replay_id": "REP-001",
  "decision_id": "DEC-001",
  "observation_chain": ["OBS-001"],
  "evidence_chain": ["EV-001"],
  "recommendation_chain": ["REC-001"],
  "validation_chain": ["VAL-001"]
}
```

# Standard Response Format

All APIs should return responses using the same format.

## Success Response

```json
{
  "success": true,
  "message": "Operation completed",
  "data": {}
}
```

---

## Error Response

```json
{
  "success": false,
  "message": "Entity not found",
  "error_code": "ENTITY_NOT_FOUND"
}
```

# Authentication

All APIs must require authentication.

Supported methods:

* API Key
* JWT Token
* OAuth 2.0

Unauthenticated requests must be rejected.

---

# API Governance Rules

1. Every request must be logged.
2. Every response must be traceable.
3. Every recommendation must reference evidence.
4. Every decision must reference recommendations.
5. Every validation must reference decisions.
6. Every API call must support auditing.
7. Every API response must be replayable.
