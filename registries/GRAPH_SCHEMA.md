# Marine Intelligence Graph Schema

## Purpose

The Marine Intelligence Graph is the central knowledge structure of the Marine Intelligence System.

Its purpose is to connect operational knowledge, infrastructure, environmental conditions, logistics information, risks, evidence, recommendations and decisions into a single explainable intelligence network.

Instead of treating data as isolated records, the graph treats every object as part of an interconnected operational ecosystem.

This allows the system to answer:

* What is happening?
* Why is it happening?
* What caused it?
* What is affected?
* What should be done?
* Why should that action be taken?
* What evidence supports it?

Every recommendation generated by NICAI must be traceable through this graph.

---

Traditional Systems:

# Graph Structure

The graph consists of:

1. Nodes (Entities)
2. Edges (Relationships)
3. Evidence Links
4. State Links
5. Temporal Links
6. Validation Links

Every intelligence output is derived from these components.

# Validation Layer

SVACS validates graph outputs.

Validation checks:

* Evidence exists
* Relationships are valid
* Confidence is justified
* Source authority is trusted
* Recommendation is explainable

No recommendation can bypass validation.

---

# Replay Layer

Every graph change is recorded.

Example:

Water Level Changed

Old Value:

5.8m

New Value:

4.9m

Timestamp:

2025-06-01 10:00

Reason:

Telemetry Update

Replay allows operators to reconstruct events.
